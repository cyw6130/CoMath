1 概览

lark-cli 的定位与核心能力

Note

🚀 lark-cli 是飞书官方推出的命令行工具，让你可以通过终端或脚本直接操作飞书生态中的各种资源，无需打开浏览器。

lark-cli 的核心定位：

  • 官方出品：由飞书开放平台团队维护，API 兼容性有保障，持续更新迭代
  • 50+ 资源覆盖：文档（Docs）、表格（Sheets）、多维表格（Bitable）、日历（Calendar）、即时消息（IM）、云盘（Drive）、通讯录（Contact）等飞书核心资源全部可用
  • 双身份模式：支持 bot（应用身份）和 user（用户身份）两种认证模式，分别适合自动化场景和个人操作场景
  • AI Agent 友好：命令行接口天然适合被 Agent 通过 shell 调用，返回 JSON 格式结果，便于程序化处理
Note

🤖 对于 AI Agent 开发者来说，lark-cli 是连接大模型与飞书协作生态的最佳桥梁 —— Agent 可以读取文档内容作为上下文，也可以将生成结果写回飞书文档，实现真正的 AI + 协作闭环。

2 安装与认证

安装方式、身份模式与权限配置

安装方式

通过 npm 全局安装 lark-cli：

npm install -g @larksuite/cli

验证安装成功：

lark-cli --version

身份模式

lark-cli 支持两种身份模式，适用于不同的使用场景：

特性 Bot（应用身份） User（用户身份）
适用场景 自动化、CI/CD、无人值守 个人操作、交互式使用
认证方式 App ID + App Secret OAuth 授权登录
权限范围 由应用配置的 Scopes 决定 继承用户的实际权限
典型命令 –as bot –as user

Bot 身份配置

  1. 前往 飞书开放平台 创建企业自建应用
  2. 记录 App ID 和 App Secret
  3. 在应用的「权限管理」页面开通所需的 OAuth Scopes（如 docs:doc:read、docs:doc:write 等）
  4. 配置认证信息并登录
lark-cli auth login --provider feishu --method app-id \
  --app-id "cli_xxxxx" \
  --app-secret "xxxxxx"

User 身份配置

用户身份通过 OAuth 流程授权，浏览器会打开飞书登录页面：

lark-cli auth login --provider feishu
Note

💡 Bot 身份需要管理员审批应用权限；User 身份需要在飞书开放平台配置好 OAuth 回调地址。建议开发调试阶段使用 User 身份，部署上线后切换为 Bot 身份。

3 文档操作

创建、读取、编辑飞书文档的完整流程

核心命令概览

操作 命令 说明
创建文档 docs +create 在指定文件夹下创建新文档
读取文档 docs +fetch 获取文档内容，支持返回 block ID
追加内容 docs +update --command append 在文档末尾追加新内容
插入内容 docs +update --command block_insert_after 在指定 block 之后插入内容
替换内容 docs +update --command block_update 更新指定 block 的内容

读取文档

获取文档内容及所有 block 的 ID（后续编辑需要用到）：

# 基础读取
lark-cli docs +fetch --api-version v2 --as bot --doc "DOC_TOKEN"

# 获取详细的 block ID（编辑文档时必须）
lark-cli docs +fetch --api-version v2 --as bot --doc "DOC_TOKEN" --detail with-ids

创建文档

lark-cli docs +create --api-version v2 --as bot --folder "FOLDER_TOKEN" --title "新文档标题"

插入内容

在指定 block 之后插入新内容，支持丰富的 XML 标签：

lark-cli docs +update --api-version v2 --as bot \
  --doc "DOC_TOKEN" \
  --command block_insert_after \
  --block-id "TARGET_BLOCK_ID" \
  --content '<p>这是一段新内容</p>'

支持的 XML 标签

  • 文本：<p> 段落、<h1>-<h9> 标题
  • 列表：<ul>/<ol>/<li> 有序/无序列表
  • 表格：<table>/<thead>/<tbody>/<tr>/<th>/<td>
  • 富组件：<callout> 高亮块、<grid>/<column> 分栏布局
  • 代码：<pre lang="bash" caption="说明"><code>...</code></pre>
  • 链接：<a type="url-preview" href="URL">标题</a>
  • 分隔线：<hr/>
Note

💡 使用 --detail with-ids 获取的 block ID 是精确编辑文档的关键。每个 block 都有唯一 ID，可以用 block_insert_after 在其后插入新内容，或用 block_update 替换其内容。

4 其他资源

表格、多维表格、日历、消息等操作

除了文档之外，lark-cli 还覆盖了飞书生态中的大部分资源类型。以下是主要资源的支持情况：

资源类型 命令前缀 核心能力
文档 docs 创建、读取、编辑文档；插入媒体、评论
表格 sheets 读写单元格、设置公式、管理样式、工作表操作
多维表格 bitable 记录 CRUD、视图管理、字段配置、自动化
日历 calendar 日程创建/查询/更新、会议室预约
即时消息 im 发送/读取消息、群组管理、消息卡片
云盘 drive 文件上传/下载、权限管理、文件夹操作
通讯录 contact 用户信息查询、部门结构、成员管理

快速示例

# 读取表格数据
lark-cli sheets +fetch --api-version v2 --as bot --sheet "SHEET_TOKEN"

# 查询多维表格记录
lark-cli bitable +query --api-version v2 --as bot --table "TABLE_TOKEN"

# 发送消息
lark-cli im +send --as bot --chat "CHAT_ID" --content "Hello from lark-cli!"

# 上传文件到云盘
lark-cli drive +upload --as bot --file "./report.pdf" --folder "FOLDER_TOKEN"
Note

🔗 完整的命令参考请访问 飞书开放平台文档，每个资源都有详细的 API 说明和示例。

5 在 Agent 中使用

将 lark-cli 集成到 AI Agent 工作流

lark-cli 最大的优势之一是天然适配 AI Agent 的工作流。Agent 可以通过 shell 调用 lark-cli 完成各种飞书操作，无需额外的 SDK 集成。

集成模式

  • Shell 调用：Agent 通过 subprocess 执行 lark-cli 命令，获取标准输出
  • JSON 输出：所有命令均返回结构化 JSON，可用 jq 等工具解析
  • 错误处理：非零退出码表示失败，stderr 包含错误详情

典型 Agent 工作流示例

import subprocess, json

def fetch_doc(doc_token):
    """读取飞书文档内容"""
    result = subprocess.run(
        ["lark-cli", "docs", "+fetch",
         "--api-version", "v2", "--as", "bot",
         "--doc", doc_token, "--detail", "with-ids"],
        capture_output=True, text=True
    )
    return json.loads(result.stdout)

def append_to_doc(doc_token, block_id, xml_content):
    """在指定 block 后插入内容"""
    result = subprocess.run(
        ["lark-cli", "docs", "+update",
         "--api-version", "v2", "--as", "bot",
         "--doc", doc_token,
         "--command", "block_insert_after",
         "--block-id", block_id,
         "--content", xml_content],
        capture_output=True, text=True
    )
    return json.loads(result.stdout)

# 使用示例

身份选择建议

场景 推荐身份 原因
CI/CD 自动化 Bot 无需人工交互，Token 持久化
定时任务 Bot 无人值守，长期稳定运行
交互式调试 User 继承个人权限，方便验证
权限受限资源 User 以用户身份访问其有权限的资源
Important

🔑 Bot 身份下，Agent 操作的文档所有者是应用而非个人。如果需要让文档出现在某人的「与我共享」中，需要额外调用权限接口添加协作者。

6 FAQ

常见问题与解决方案

以下是使用 lark-cli 过程中最常见的问题及解决方案：

Q1: 认证失败怎么办？

  • 确认 App ID 和 App Secret 是否正确，注意不要有多余的空格
  • 检查应用是否已发布（企业自建应用需要管理员审核通过）
  • 尝试重新登录：lark-cli auth login --provider feishu
  • 清除缓存后重试：lark-cli auth logout 然后重新 login

Q2: 提示权限不足（Permission Denied）？

  • Bot 身份：前往飞书开放平台，在应用的「权限管理」中开通对应的 API Scopes，并让管理员审批
  • User 身份：确认你的飞书账号对目标资源有读写权限
  • 部分高级权限（如通讯录写入）需要企业管理员在后台开通

Q3: API 版本选 v1 还是 v2？

  • 新项目一律使用 --api-version v2，v2 是当前主推版本，功能更完善
  • v1 仅用于兼容旧脚本，新功能只在 v2 上提供

Q4: 遇到速率限制（Rate Limit）？

  • 飞书 API 有调用频率限制，Bot 身份的限制通常比 User 身份更宽松
  • 建议在 Agent 中加入重试逻辑，遇到 429 状态码时等待 1-2 秒后重试
  • 批量操作建议使用官方提供的批量接口，而非循环调用单条接口
Warning

🆘 如果以上方案都无法解决问题，可以前往 飞书开放平台 查看官方文档或提交工单。社区中也有大量 lark-cli 的使用案例可以参考。
