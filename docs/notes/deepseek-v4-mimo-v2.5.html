介绍DeepSeek V4与MiMo V2.5国产大模型
Note

💡 DeepSeek V4 和 MiMo V2.5 是 2026 年最值得关注的国产大模型。DeepSeek V4 以极致性价比著称；MiMo V2.5 是小米推出的 310B MoE 多模态模型，原生支持图像/视频/文档理解，1M 上下文窗口。

1 DeepSeek V4

模型矩阵、API 接入与最佳实践

Note

🔗 平台：platform.deepseek.com  |  API：api.deepseek.com  |  兼容 OpenAI API 格式

1.1 模型矩阵

模型 定位 特点
deepseek-v4-flash 性价比首选 快速响应、低成本，适合高并发场景
deepseek-v4-pro 最强能力 深度推理、复杂任务，代码和数学推理能力突出
Note

💡 旧模型名 deepseek-chat 和 deepseek-reasoner 将于 2026/07/24 废弃，请尽快迁移到 v4 系列模型。

1.2 API 接入

DeepSeek V4 完全兼容 OpenAI API 格式，可以直接使用 OpenAI SDK 通过修改 base_url 接入。以下是两种常见调用方式：

1.2.1 curl 示例

curl https://api.deepseek.com/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY" \
  -d '{
    "model": "deepseek-v4-flash",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "解释量子纠缠的基本原理"}
    ],
    "temperature": 0.7,
    "max_tokens": 4096
  }'

1.2.2 Python 示例

from openai import OpenAI

client = OpenAI(
    api_key="your-deepseek-api-key",
    base_url="https://api.deepseek.com"
)

response = client.chat.completions.create(
    model="deepseek-v4-flash",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "用 Python 实现一个 LRU Cache"}
    ],
    temperature=0.7,
    max_tokens=4096,
)

print(response.choices[0].message.content)

1.3 最佳实践

  • 模型选择：日常对话和简单任务用 deepseek-v4-flash，需要深度推理或复杂代码生成时用 deepseek-v4-pro
  • 流式输出：交互式场景建议开启 stream: true，提升用户体验
  • 上下文管理：虽然支持 128K 上下文，但建议控制在 32K 以内以获得最佳性能和性价比
  • 温度参数：代码生成建议 0.0-0.3，创意写作 0.7-1.0

2 MiMo V2.5

310B MoE 多模态模型、平台接入与能力详解

Note

🔗 官网：mimo.xiaomi.com/mimo-v2-5  |  平台：platform.xiaomimimo.com  |  AI Studio：aistudio.xiaomimimo.com

2.1 核心参数

Note

🧠 架构：310B MoE（Mixture of Experts） 激活参数：15B（推理时仅激活约 5% 参数） 训练数据：48T tokens

Note

💡 原生多模态：图像 / 视频 / 文档理解 上下文窗口：1M tokens 开源协议：Apache 2.0

2.2 模型变体

变体 价格倍数 定位 适用场景
MiMo-V2.5 1x 标准版 日常多模态任务，性价比均衡
MiMo-V2.5-Pro 2x 增强版 复杂推理、高精度多模态理解
MiMo-V2.5-Flash 0.25x 轻量版 高并发、低延迟、简单任务

2.3 能力亮点

  • 图像理解：支持图表解析、OCR 文字识别、视觉问答，可直接理解扫描件和手写笔记
  • 视频理解：原生处理视频流，支持长视频摘要、关键帧提取、动作识别
  • 文档理解：1M 上下文窗口可一次性处理整本书或超长报告，支持结构化信息提取
  • 代码能力：在代码生成和理解任务上表现优秀，支持多语言编程
  • 长推理链：利用 1M 上下文进行多步复杂推理，适合 Agent 工作流

2.4 接入方式

import requests

url = "https://platform.xiaomimimo.com/v1/chat/completions"
headers = {
    "Authorization": "Bearer YOUR_MIMO_API_KEY",
    "Content-Type": "application/json"
}
data = {
    "model": "MiMo-V2.5",
    "messages": [
        {"role": "user", "content": [
            {"type": "text", "text": "请分析这张图片中的数据趋势"},
            {"type": "image_url", "image_url": {"url": "https://example.com/chart.png"}}
        ]}
    ],
    "max_tokens": 4096
}

response = requests.post(url, json=data, headers=headers)
print(response.json()["choices"][0]["message"]["content"])

2.5 开源资源

  • HuggingFace：XiaomiMiMo/MiMo-V2.5 — 模型权重、文档和评估结果
  • AI Studio：aistudio.xiaomimimo.com — 在线体验平台
  • 协议：Apache 2.0，可商用部署
